# Spec: mvp-0007 — Recording UX + Transcript Isolation + Cleanup

## Goal
在“一个项目多次录音”的真实使用场景下：
- 录音、播放、转写、改写的结果严格归属到当前项目，避免错乱。
- 单条录音失败（如静音/无语音）不影响其它录音的转写与汇总。
- 录音列表以“逐条显示”的方式呈现，支持播放、单删、多选批量删除。
- 录音中提供清晰的“正在录音”视觉反馈（时间 + 波形/电平）。
- 清理“项目残留数据”：删除项目/录音时同步清理云端对象与本地缓存。

## Scope
### In
- 后端
  - 录音列表接口返回“最新转写文本”（按录音维度）以便页面直接展示。
  - `DELETE /api/recordings/:id`：删除录音（含 R2 对象 + DB 记录）。
  - `POST /api/recordings/bulk-delete`：批量删除录音（含 R2 对象 + DB 记录）。
  - `DELETE /api/projects/:id`：删除项目时 best-effort 清理该项目下 R2 录音对象。
  - `/api/stt/transcribe` 支持传 `recordingId`，成功时落库到 `transcripts`。
- 前端
  - 录音列表改为“逐条列表展示”（不再使用下拉选择）。
  - 每条录音：播放、转写、单条删除；前置复选框；顶部支持批量删除。
  - “转写全部”容错：逐条转写，失败只标记该条错误，流程继续。
  - “汇总到文本”：将本项目所有录音的最新转写结果按时间顺序合并到项目草稿文本。
  - 录音中显示红点 + 波形 + 计时。
  - 提供“清理本地缓存”按钮，清理本地旧缓存（转写/改写/当前项目指针）。

### Out
- 后台异步队列/并发转写、进度持久化到 jobs 表、录音剪辑/编辑器。

## Acceptance
- 在同一项目中连续录音多次：列表按时间逐条显示，每条可直接播放。
- 对包含静音的录音执行“转写全部”：其它录音仍成功转写；静音条目显示错误信息。
- 录音删除（单条/批量）会从列表消失，且不会影响其它录音；删除项目后其录音不再残留（R2 best-effort 清理）。
- 刷新页面/换设备后：已成功转写的录音仍能在录音列表中看到“转写文本”（来自后端落库）。
