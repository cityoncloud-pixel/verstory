# Spec: mvp-0003 — Mock Providers (Transcription + Story Polish)

## Goal
在无网络/无密钥条件下，提供可替换的 mock 转写与 mock 故事整理能力，并打通 UI 触发生成与展示。

## Scope
### In
- Provider 接口：
  - `TranscriptionProvider.transcribe(segments) -> { perSegment, mergedText }`
  - `StoryPolishProvider.polish(mergedText, rules) -> { storyText, rulesSummary }`
- 默认 mock 实现：
  - 输出必须确定性（相同输入 → 相同输出）
  - 文本内容遵循“尊重原意”不可变规则（不虚构、不改事实；仅轻度整理）
- UI 串联：
  - 点击“转写（mock）”生成并展示原始转写全文
  - 点击“生成故事（mock）”生成并展示整理后故事
- 持久化：
  - 最小要求：刷新页面后仍能恢复转写与故事文本（本任务允许使用 localStorage 持久化，避免 IndexedDB schema 变更）

### Out
- 真实转写/真实 LLM 接入
- 复杂编辑器（段落编辑、版本对比等）

## Acceptance / Verification
- 离线确定性：同一项目、同一片段集合，重复点击/刷新后生成文本一致
- 可观察：UI 同时显示“原始转写”和“整理后故事”，并展示整理原则摘要
