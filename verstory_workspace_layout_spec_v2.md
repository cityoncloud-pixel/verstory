# Verstory 工作台重构规格（Workspace Layout Spec v2）

## 文档目的
本规格用于替换旧版工作台布局，建立稳定、统一、跨端一致的工作流骨架。

适用端：
- Mobile
- Tablet
- Desktop

目标：
将工作台从“长页面堆叠模式”重构为“稳定工作流模式”。

---

# 一、问题复盘（复审结论）

## 1. 顶部 Hero 区过大
问题：
- 占用首屏高度过多
- 抢占任务入口

修正：
- 删除 Hero 区

审核：PASS

---

## 2. 右侧纵向导航错误
问题：
- 占宽
- 打断阅读路径
- 危险操作暴露过多

修正：
- 收纳进顶部菜单

审核：PASS

---

## 3. 三阶段内容混合堆叠
问题：
- 页面过长
- 用户迷失

修正：
- Tab切换 = 内容切换

审核：PASS

---

## 4. 核心导航不稳定
问题：
用户滚动后失去定位。

修正：
固定顶部 + 固定底部。

审核：PASS

---

# 二、新工作台架构（审核通过版）

统一结构：

顶部固定项目栏  
中间唯一内容区  
底部固定流程栏

结构：

顶部项目栏（固定）
↓
中间内容区（滚动）
↓
底部流程栏（固定）

审核结论：

PASS

无逻辑冲突。

---

# 三、顶部固定栏规格

高度：
56~64px

固定：
position: sticky

## 结构

左侧：
项目选择器

中间：
当前项目名

右侧：
更多菜单（⋮）

结构：

[项目 ▼]        ⋮

## 项目菜单

切换项目  
新建项目  
修改项目名  
删除项目  
设置  
退出登录  

风险控制：

删除项目必须二次确认。

审核：

PASS

---

# 四、底部固定流程栏

固定：
position: fixed

高度：
64px

包含：

录音  
转写  
整理稿  

规则：

只切换阶段，不滚动定位。

审核：

PASS

---

# 五、中间内容区规则

唯一滚动区。

禁止多滚动容器。

高度计算：

viewport - topBar - bottomBar

规则：

当前只渲染一个阶段内容。

## 录音阶段

显示：

- 开始录音
- 上传录音
- 录音列表

## 转写阶段

显示：

- 转写列表
- 原始文本
- 校正文本

## 整理阶段

显示：

- 整理风格
- 整理结果
- 导出按钮

审核：

PASS

---

# 六、录音阶段结构

顶部操作：

开始录音  
上传录音  

录音中状态：

暂停录音  
停止录音  

管理录音：

进入多选模式。

删除：

单条删除、多选删除均需确认。

审核：

PASS

# 七、转写阶段结构

列表：

按时间排序。

操作：

开始转写  
重新转写  
编辑校正  
保存  
复制  

支持：

多录音合并转写。

审核：

PASS

---

# 八、整理阶段结构

风格选择：

忠实整理  
自传口吻  
访谈稿  
家族记录  

按钮：

生成整理稿  
重新整理  
导出  

默认展示：

整理稿。

可展开：

原始稿。

审核：

PASS

# 九、跨端规则

## 手机
顶部固定  
底部固定  
中间滚动  

## 平板
结构不变  
布局变宽  

## 桌面
结构不变  
允许更宽内容区  

禁止改变信息架构。

审核：

PASS
