# 口述整理工具（Oral Story）交互技术规格书（Interaction Spec v1.0）

## 1. 文档目标

本文档定义「口述整理工具」的完整交互规格，用于指导 Codex/Cursor 进行产品实现。

产品目标：

帮助用户（尤其是中老年用户）完成：

**口述 → 录音 → 转写 → 校正 → 整理成文 → 导出保存**

核心原则：

- 极简操作
- 大按钮优先
- 单路径推进
- 避免误操作
- 支持长期积累型内容创作

---

# 2. 产品定位

不是录音软件。

不是纯转写工具。

而是：

**人生经历记录整理工具**

适合：

- 回忆录
- 自传
- 家族口述史
- 人生故事整理
- 经历记录

---

# 3. 核心用户画像

## 用户类型A（主要）

年龄：

50~75岁

特点：

- 不熟悉复杂软件
- 不理解技术概念
- 容易误触
- 操作耐心有限

需求：

快速录音并形成文章。

---

## 用户类型B（辅助）

家庭成员整理协助者。

特点：

- 帮助长辈整理内容
- 可做文本修订

# 6. 页面规格

目标：

功能：

# 7.5 删除机制

原则：

流程：

```text

```

不新建项目。

# 14. UI规范

原则：

大字号。

大按钮。

高对比。

少按钮。

---

按钮最少：

---

单页面原则：

用户当前页面只做一件事。

# 18. 开发约束

禁止：

将开发者功能暴露给普通用户。

禁止：

将 API 日志放在主界面。

禁止：

将模型选择暴露在主流程中。

必须：

保持三步主流程稳定：

```text
录音 → 转写 → 整理
```

---

# 19. 验收标准

验收通过标准：

- 长辈可独立完成录音
- 可独立转写
- 可独立生成整理稿
- 无误删风险
- 无迷路操作路径
