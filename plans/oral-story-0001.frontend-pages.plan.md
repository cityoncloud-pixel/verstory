# Plan: oral-story-0001 — Frontend Pages (M1)

## 1) Information architecture (pages)
- `/login`：登录/注册/找回密码（大字号大按钮）
- `/projects`：项目列表（新建/进入/删除）
- `/projects/:id`：项目工作台（只做一件事：沿着主流程推进）
  - Tab1：录音（录音/上传/播放/删除/批量删）
  - Tab2：转写（单条转写 / 转写全部 / 查看转写结果）
  - Tab3：整理（校正稿编辑、生成整理稿、导出）
- `/settings`（可选，M1 可弱化）：provider 状态、缺 key 提示、清理缓存

## 2) Page-level UX rules
- 每页主按钮 ≤ 3
- 删除操作：二次确认 + 明确说明影响范围（只删当前项目）
- 所有长耗时：显示明确状态（转写中/整理中/失败可重试）

## 3) Project workspace (“三步主流程”)
### 录音 Tab
- 主按钮：开始/停止（或开始/暂停/停止，择一保持极简）
- 次按钮：上传录音
- 列表：逐条显示（播放、删除、选择框）
- 批量：全选/删除所选（带确认）

### 转写 Tab
- 主按钮：转写全部（逐条容错，不因单条失败中断）
- 单条：转写 / 重试
- 展示：每条录音的“最新转写文本”

### 整理 Tab
- 输入：合并后的“校正稿”（可编辑保存）
- 操作：生成整理稿（至少 2 种风格：口述整理/回忆录）
- 导出：下载 Markdown 或复制到剪贴板（至少一种）

## 4) Client data strategy
- 以服务端为准（DB + 对象存储）
- 本地只做：
  - 当前 projectId 的 UI 状态缓存
  - 录音中临时缓冲（上传前）

## 5) Acceptance checklist
- 创建项目 → 录音 2 条 → 转写全部 → 校正保存 → 整理成文 → 导出
- 删除录音（单条/批量）与删除项目均有确认，且不会删到其它项目
