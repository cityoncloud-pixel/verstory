# Plan: mvp-0007 — Recording UX + Transcript Isolation + Cleanup

## 1) 数据与接口
- 扩展 `GET /api/projects/:projectId/recordings`：带出最新 transcript（text/provider/model）
- `DELETE /api/recordings/:id`：删除 DB 记录并删除 R2 对象
- `POST /api/recordings/bulk-delete`：批量删除（DB + R2）
- `DELETE /api/projects/:id`：项目删除前 best-effort 清理其录音 R2 对象
- 扩展 `/api/stt/transcribe`：支持传 `recordingId`，成功后写入 `transcripts`

## 2) 前端交互
- 录音列表改为逐条展示（含：复选框、播放、转写、单删、错误提示、转写结果）
- 新增“全选/取消全选/删除所选/刷新”
- 新增“转写全部”：逐条容错；成功/失败计数；失败不打断
- 新增“汇总到文本”：按时间顺序合并所有转写文本并保存草稿
- 录音中增加红点 + 波形 + 计时
- 新增“清理本地缓存”按钮

## 3) 验证
- 录 3 条（含 1 条静音）→ 转写全部 → 检查列表与汇总文本
- 批量删除 2 条 → 刷新列表与播放/转写按钮状态
- 删除项目 → 确认项目与录音消失（R2 清理 best-effort）
